> Introduction

TypeScript is a superset of JavaScript that introduces static typing to the language. This means that you can explicitly declare the types of variables, function parameters, and return values, allowing for better code organization and catching errors before runtime. Understanding how to define basic types in TypeScript is essential to harness the full power of the language.

> Problem

JavaScript is dynamically typed, which can lead to unexpected runtime errors when variables hold unexpected values. For example, a variable that is expected to be a string might suddenly hold a number or even be undefined. These errors can be hard to debug and maintain in large codebases, impacting the overall development process.

&nbsp;

> Solution

In TypeScript, you can define basic types using a combination of built-in primitive types and custom types.

# Here are some of the essential basic types:

1. Number

```ts
let amount: number = 20;
```

2. String

```ts
let course: string = 'Typescript';
```

3. Boolean

```ts
let isLoading: boolean = true;
```

4. Any

```ts
let book;
```

```ts
function render(document: any) {
  console.log(document);
}
```

5. Array

```ts
let arrayOfNumber: number[] = [1, 2, 3];
let arrarOfString: string[] = ['apple', 'orange'];
let arrayOfAny = [];
```

6. Tupple

```ts
let tupples: [number, string] = [1, 'ayam'];
```

7. Enum

```ts
const enum Size {
  Small = 1,
  Medium,
  Large,
}

let mySize: Size = Size.Medium;
console.log(mySize); // 2
```

8. Object

```ts
let employee: {
  readonly id: number;
  name: string;
  retire: (date: Date) => void;
} = {
  id: 1,
  name: 'bayu',
  retire: (date: Date) => {
    console.log(date);
  },
};
```

9. Function

```ts
// without parameter
function calculateTax(): number {
  return 10;
}

// number parameter
function calculateTax2(income: number): number {
  if (income > 50000) return income * 2;
  return income * 1;
}

// number multiple parameter
function calculateTax3(income: number, taxYear: number): number {
  if (taxYear > 2022) return income * 2;
  return income * 1;
}
calculateTax3(10000, 2021);

// multiple & optional parameter
function calculateTax4(income: number, taxYear?: number): number {
  if ((taxYear || 2022) > 2022) return income * 2;
  return income * 1;
}
calculateTax4(10000);

// multiple, optional & default value parameter
function calculateTax5(income: number, taxYear = 2022): number {
  if (taxYear > 2022) return income * 2;
  return income * 1;
}
calculateTax5(10000);
```

> Summary

Defining basic types in TypeScript is crucial for writing maintainable and error-free code. By specifying types explicitly, you can catch errors early in the development process and improve code readability. TypeScript provides a range of built-in primitive types, as well as the flexibility to create custom types, enabling you to create structured and organized applications.

In this blog post, we've covered the importance of defining basic types, the challenges posed by JavaScript's dynamic typing, and the solutions TypeScript offers. Armed with this knowledge, you're ready to start using basic types effectively in your TypeScript projects.
